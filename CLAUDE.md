# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this repository is

This is not a software project — it's a collection of **Claude Skills** (Markdown-based, instruction-following skill definitions) for academic reading and text study, written in Portuguese. There is no build, lint, or test tooling: the only "code" is the natural-language instructions inside each `SKILL.md`, which Claude interprets and follows at runtime when the skill is invoked.

## Repository structure

Each skill lives in its own top-level directory and is self-contained:

```
triagem-de-leitura/
├── SKILL.md
└── README.md
leitura-analitica-severino/
├── SKILL.md      # the skill definition (frontmatter + instructions Claude follows)
└── README.md      # human-facing description, for people browsing the repo
tutor-de-texto/
├── SKILL.md
└── README.md
leitura-guiada/
├── SKILL.md
└── README.md
leitura-livro-didatico-sem-digitalizacao/
├── SKILL.md
└── README.md
leitura-cientifica-keshav/
├── SKILL.md
└── README.md
leitura-literaria-cosson/
├── SKILL.md
└── README.md
```

- `SKILL.md` files start with YAML frontmatter (`name`, `description`). The `description` field is what a Claude Code instance uses to decide *when* to auto-invoke the skill — it must exhaustively cover the phrasings a user might use to trigger it, including indirect ones that don't name the skill explicitly.
- A skill is installed by copying its folder (containing `SKILL.md`) into the user's Claude skills directory, or by packaging the folder as a `.skill`/`.zip`.
- The root `README.md` is the index of skills in this kit; each skill's own `README.md` is a shorter, install-focused description of that one skill. Keep both in sync with the corresponding `SKILL.md` when a skill's behavior changes.

## The seven skills

- **`triagem-de-leitura/`** — Not a reading skill itself: it's the entry point for a reader who has a text but doesn't know which of the other six skills fits. Runs a short, one-question-at-a-time intake (text type + medium, then reading-experience level, then — for generic academic/technical text only — reading goal), then presents the relevant options with a stated recommendation and lets the reader pick before handing off. Reading-experience level matters even when the text type "obviously" points to one fixed-method skill (e.g., a scientific paper doesn't automatically mean `leitura-cientifica-keshav` if the reader has never read one independently — `tutor-de-texto` may surface as a starting point instead). Triggers only on genuine indecision ("não sei qual skill usar" and similar); if the request already names a method, author, or text type clearly, the request should route straight to the specific skill instead, not through this one. Never performs any reading, analysis, or fichamento itself, and never produces a downloadable file — it only recommends and hands off.
- **`leitura-analitica-severino/`** — Produces a formal analytical document (a *fichamento*) from an academic text, following Antônio Joaquim Severino's five-stage method (análise textual, temática, interpretativa, problematização, síntese pessoal). Output is always a downloadable `.md` file, with an optional CmapTools-format `.txt` concept map as an extra deliverable. Supports six output formats (bibliographic record, citations record, indicative/informative summary per NBR 6028, full review, or the full five-stage analytical record).
- **`tutor-de-texto/`** — Conducts an interactive, conversational, point-by-point tutoring session over an academic/technical text until the user can explain its core (theme, problem, thesis, conclusion) in their own words. Focused on the *process* (mandatory comprehension checks between concepts, never advancing without a real user response), not just the final document. Reorganizes the content into its own pedagogical sequence (not necessarily the text's own order). Also outputs a `.md` study roadmap at the end.
- **`leitura-guiada/`** — Accompanies the reader through a text sequentially, section by section, in the text's own original order — centered on the reader and their context, not on a method of analysis (that's `leitura-analitica-severino`) nor on a reorganized pedagogical sequence (that's `tutor-de-texto`). Runs a brief context interview, a pre-reading survey of what the reader already knows, confirms with the reader how much of the text to cover (whole text or a scoped subset relevant to their stated purpose), checks comprehension section by section (explaining prerequisites on demand), and ends with a session report recommending further reading (foundational + interest-based). Always delivered as a downloadable `.md` file.
- **`leitura-livro-didatico-sem-digitalizacao/`** — Accompanies a reader through a printed textbook chapter when **no digitized version of the text exists at all**: Claude never has access to the chapter's actual content, only to what the reader reports about it during the session. Uses page-anchored questions (page numbers, layout elements, order of concepts) as evidence of real reading — the principle being that faking the session should be harder than actually reading — while explicitly prioritizing pedagogy over that verification mechanism whenever the two conflict. Always delivered as a downloadable `.md` session report meant to be attached as coursework (e.g., in an LMS such as Moodle).
- **`leitura-cientifica-keshav/`** — Accompanies the reading of a **single scientific paper**, following S. Keshav's three-pass method (*How to Read a Paper*, ACM SIGCOMM CCR, 2007): a first pass for panorama (the "5 Cs" — Category, Context, Correctness, Contributions, Clarity), a second pass for content comprehension (with attention to figures/tables and a personal summary of the central argument), and a third, critical-reading pass whose depth is calibrated by the reader's declared experience level (beginner / intermediate / advanced-researcher). Unlike the other five skills, Claude has full access to the (already digitized) text throughout, which makes the anti-"ler sem ler" discipline here structurally different: each pass must be performed by the reader themselves, with Claude only checking the reader's report against the real text, never summarizing or analyzing the paper in the reader's place. Scoped to one paper at a time — not a literature-review/multi-source skill. Always delivered as a downloadable `.md` session report.
- **`leitura-literaria-cosson/`** — Accompanies the study of a **literary work** (poem, short story, novel, chronicle, play) using Rildo Cosson's *letramento literário* (*Letramento Literário: Teoria e Prática*, 2009): the *sequência básica* (motivação, introdução, leitura, interpretação) by default, or the *sequência expandida* as an opt-in deeper path (same first three steps, then primeira interpretação → contextualização → segunda interpretação → expansão). Opens by asking whether the reader already knows Cosson's method — if so, present the two sequence options directly and ask whether the Motivação step is even needed (a teacher may have already run it in person, which is often why the reader shows up with the text already in hand); if not, explain both sequences in plain terms before letting the reader choose. The central discipline is different from the other five: the risk isn't Claude reading *instead of* the person, it's Claude handing over "the meaning" of the work instead of letting the person's own interpretation form — factual/verifiable information (biography, historical context, literary movement) is only allowed in Introdução/Contextualização, never as a substitute for the reader's interpretive judgment. Always delivered as a downloadable `.md` session report.

Of the six actual reading skills, the first three are designed to work on user-supplied text only (pasted, uploaded, or linked); `leitura-livro-didatico-sem-digitalizacao` is the complementary case for when no digitized text exists — if a request implies only a physical, undigitized book, route to that skill instead of the others. `leitura-cientifica-keshav` and `leitura-literaria-cosson` also work on user-supplied (digitized) text, but are narrowly scoped by genre (scientific paper vs. literary work) rather than being generic academic-text companions. All six explicitly forbid inventing content about the source, whether that source is the text actually provided or the reader's own account of a chapter Claude cannot see. `triagem-de-leitura` sits outside this grouping — it never touches source content, so the anti-fabrication constraint doesn't apply to it the same way; its own risk is confidently routing to the wrong skill, which is why its trigger phrasing is deliberately narrow (see above).

## Working on skill definitions

When editing a `SKILL.md`:
- Preserve the distinction between the seven skills' purposes: `triagem-de-leitura` is about *routing* — it never reads, analyzes, or interprets a text itself, only asks a short sequence of questions and points to the right skill; `leitura-analitica-severino` is about the *deliverable* (a structured record for later reference); `tutor-de-texto` is about the *process* of reaching comprehension via its own pedagogical reorganization of the content; `leitura-guiada` is about following the *text's own sequential order*, reader-centered rather than method- or pedagogy-centered; `leitura-livro-didatico-sem-digitalizacao` is about generating *evidence of real reading* when Claude has no access to the source at all — reader-centered like `leitura-guiada`, but built around a fundamentally different constraint (no source text, only the reader's report); `leitura-cientifica-keshav` is about *triage and critical reading efficiency* for a single scientific paper via three cumulative re-readings of increasing depth, calibrated by the reader's experience level — not a fichamento, not a generic sequential companion, not a multi-source review; `leitura-literaria-cosson` is about *forming the reader's own interpretive capacity* for a single literary work via Cosson's sequence, where the anti-shortcut discipline is about withholding "the meaning" rather than withholding the text. Don't blur them.
- The `description` frontmatter field drives auto-invocation matching — when adding new trigger phrasings or use cases to a skill's behavior, update `description` too, not just the body instructions.
- All six reading skills mandate delivering results as a downloadable file (never only as inline chat text) and forbid fabricating source content, biographical data, or quotes not present in the original text — preserve these constraints in any edits. `triagem-de-leitura` is the one exception to the downloadable-file rule: it only recommends and hands off, so it never produces a session file of its own. `leitura-livro-didatico-sem-digitalizacao` takes the fabrication constraint furthest: since Claude never sees the chapter, absolutely everything about its content must come from the reader's own report, never invented or guessed. `leitura-cientifica-keshav` takes a different tack on the same risk: Claude *does* have the text, so the constraint is procedural — never do a pass on the reader's behalf, only verify their report against the source. `leitura-literaria-cosson` takes yet another tack: Claude has the text, and paraphrase/factual content is allowed, but the interpretive judgment itself must always come from the reader.
- `leitura-analitica-severino` has three distinct "modos de condução" (dialogue / in-document suggestions / direct expert mode) for stages 3–5 that change *how* content for those stages is obtained but never the final document's structure — keep this invariant intact when modifying that section.
- `leitura-livro-didatico-sem-digitalizacao` explicitly prioritizes pedagogy over its own anti-fabrication verification mechanism (page-anchored questions) whenever the two conflict — don't let edits tighten verification at the expense of genuinely helping the reader.
- `leitura-literaria-cosson` defaults to the *sequência básica*; the *sequência expandida* is opt-in. The Motivação step can be skipped if the reader confirms it already happened offline (e.g., a teacher ran it in class) — don't make Motivação mandatory regardless of context. In the *sequência expandida*, session length is never a reason to rush Segunda interpretação or Expansão toward closing — the reader may explore multiple axes/relations before moving on.
- `triagem-de-leitura`'s trigger phrasing must stay narrow (genuine indecision only) — broadening its `description` to catch generic reading requests would make it compete with the other six skills' own auto-invocation and add an unwanted extra hop for readers who already know what they want. It must never simulate another skill's behavior from memory when that skill isn't installed; it should say so plainly and point to the kit README instead. Never let it skip straight from text type to a recommendation — the reading-experience question always runs first, even for the three text types (scientific paper, literary work, undigitized textbook) that map to a single fixed-method skill, since low experience can surface `tutor-de-texto`/`leitura-guiada` as a better starting point regardless of genre.
